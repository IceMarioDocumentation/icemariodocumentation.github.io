Ice Mario Documentation


About

Ice Mario is a Mario Kart Tour bot with many features focused around the competetive scene. This page will outline all the commands available in detail, and some commands will also have example use cases. You can also use the command im!help to get help on a certain module.

Ice Mario's invite link is temporarily unavailable because it has reached the Discord limit of being in 100 servers. Once the bot is verified, the invite link will be available again!

If you have any questions, want to suggest new features, would like me to add a custom command for your server, or would like to request stored data be deleted, feel free to contact me on Discord: Jimmy ツ#5440.

New Features

(Updated November 8, 2021)

Added the bold table style option. This will make the player names, player scores, and the header be in bold. For example, the top line in the table command should be !tablel bold. This will also work for styled tables, such as for CWL: !tablel cwl bold.

The table header will now be dynamically scaled based on the amount of text. If there is a lot of text, mainly due to custom titles, it will be made smaller to prevent it from being squeezed as much as possible.

Score tracker updates: You can now add or remove a specific race. For example, to remove race 5, do !u 5. To add a fifth race where the finish positions were 1, 2, 3, 4, do !a 5 1 2 3 4. By using !u # followed by !a # x1 x2 x3 x4, you are able to update a race.

1. Clan War Scheduler

Creates schedules for people to react to. Note: the following commands will only work if enable-schedule is set to true (default is true). By default, Ice Mario will ping members that have reacted 1 hour before an event as well as when an event starts. Also, each schedule will be disabled (indicated by the color black) 1 hour after the scheduled time. You can also have Ice Mario delete the schedule after 1 hour instead. This can all be adjusted by setting the Clan War Scheduler config options.

By default, anyone can create and edit schedules. If you would like to change it so that only moderators are able to, change the config value moderators-only-for-scheduler to be true.

If someone signs up for two or more events happening at the same time, Ice Mario will send a warning message about the schedule conflict.

The schedule message color will change based on the number of people that have signed up. They are as follows:

  • Red: The can list and sub list have a combined total of 2 or less people.
  • Green: There are 4 or more players on the can list.
  • Yellow: When there are almost enough people for an event.

!create <event title> <date and time> [optional restriction or preference]
Create a new schedule. You will need to provide the event title as well as the date. Optionally, you can use !createc instead to create a condensed schedule; each member is shown on one line instead of on separate lines for each reaction. optional restriction or preferance will show up in a line named Restricted, and this allows people to see what the preferance is. Reactions can be restricted so that only certain roles can react to join. If you would like this feature, feel free to contact me (Jimmy ツ#5440).

People can react to one of the three reactions on the message, and the schedule will update accordingly. It's recommended to give Ice Mario MANAGE_MESSAGES permission in your schedule channel because it will automatically remove someone's previous reaction if they switch to a different one.

!scheduleedit <schedule message link> <new title and date>
Edit an existing schedule's title, the date, or the restriction.

!scheduleadd <schedule message link> <can/sub> <member ping>
Add someone to an existing schedule manually. The second argument should be either "can" or "sub".

!scheduleremove <schedule message link> <member ping>
Remove someone from an existing schedule manually.

!schedule
Get a list of all the schedules in your server for the day (based on PT time).

!myschedule
Get a list of all the schedules you have signed up for.

!stickyschedule
Initializes a sticky schedule for a server. This will show a list of all the schedules, and it will be updated once a minute. This is most helpful if its used in its own channel. Note: you will need MANAGE_MESSAGES permission. This command can be used multiple times, but only the most recent message will be updated. You may delete the sticky schedule message to stop further updates.

Add one of the following reactions to a schedule to modify it:
📝: Edit the schedule title, date, or restriction.
❌: Enable or disable a schedule. The schedule will be black if disabled.
🔄: Switch between a condensed and uncondensed schedule format.

2. Clan War Score Tracker

Keep track of your team's score during a war. Note: the following commands will only work if enable-score-tracker is set to true (default is true). This will work for both a 4v4 and a team survival war event.

!startwar opponentTeam or
!startwar yourTeam opponentTeam
Starts a new score tracker by clearing the previous scores and sets the team names. This is equivalent to using !u all then !setteams. If the top command is used (only specify the opponent team), then yourTeam's default value is default-team-name (default value is "team1"); you can change this by updating the server config settings. If yourTeam is provided, this will override the value in the config settings.

!setteams opponentTeam or
!setteams yourTeam opponentTeam
Set the opponent team or both team names. This can be skipped if you use !startwar.

!a x1 x2 x3 x4 or
!a [optional race #] x1 x2 x3 x4
Adds the score for a race using finish positions x1 to x4. If optional race # is specified, the scores will be added for that race.

!u [optional #]
Undo a race. If optional # is provided, that race will be removed. Otherwise, it will undo the most recent race score.

Examples:

  • !u: Undo the most recent race.
  • !u 7: Undo race 7.
!u all
Clears all the race scores in the current channel

!penalty <points> <team name>
Applies a penalty to the specified team

The following commands can be used to keep track of a team survival war:

!startsurvival
team1Name, player1, player2, player3, player4
team2Name, player1, player2, player3, player4
Starts a team survival war life tracker. See below to see how to create an FFA survival tracker. You can use !ss instead of !startsurvival. There should be 3 lines total.

!startsurvival player1, player2, ..., playerN
Starts an FFA survival war life tracker. For FFA, the command and player list should all be on the same line. There can be any number of players; it is not limited to only 8 players. You can use !ss instead of !startsurvival.

!x player#
Remove a life from someone.

Note: You can use !u and !setteams during survival wars. For the team survival format, once a team has lost all their lives, Ice Mario will automatically generate a table!

3. Table maker

Make a clan war table. Note: the following commands will only work if enable-table-maker is set to true (default is true). Ice Mario's table maker can easily add team logos, specify a table style (such as for cwl), and will generate a table with twice the resolution as from gb.hlorenzi.com for a more clear image!

!table [style options]
[table options]
Team1
Player1 [us] 90
Player2 [mx] 90
...
To add team logos, use !tablel instead of !table.

[style options]: Optional style modifiers. All style options should be written on the same line as the command, separated by a space.

  • cwl: Make a CWL table
  • cyclone: Make a cyclone table
  • sw: Make a survival war table
  • mku: Make an MKU table
  • mkt: Add the MKT logo in the middle of the table. cyclone and sw will automatically add it, unless you specify mktoff
  • mktoff: Don't add the mkt logo. This option can be used to force the table maker to not add the MKT logo.
  • bold: Make the player names, player scores, and the header bold.

[Table options]: Optional table modifiers. Each table option should be on its own line, and it should start with a #.

  • #date <date>: Set the date on the header
  • #title <title>: Set a custom title to the left of the date
  • #dotalpha <val>: Set the dot transparency. The value should between 0 and 1; a value of 0 will not display the dots.
  • #dotcolor <#000000>: Set the dot color in hex format
  • #bgbright <val>: Set the background brightness if a background image is being used. Val should be between 0 and 1 where 0 is full dark and 1 is full bright.

Additional ways to customize the table:

  • To set the table background with an image, send the image at the same time you use the command. Then, the default table option values are dotalpha=0, and bgbright=0.7.
  • To set the team color, add a hex color string (#000000) to the right of the team name. This will override the colors if !tablel is used, but this will have no effect if a style option is used (such as cwl, cyclone, or mku).
  • To add a penalty to a player, subtract the penalty amount to the right of the player's score
  • To add a penalty to a team, add a new player to a new line with the name Penalty and subtract the penalty amount. For example, adding Penalty -10 will apply a penalty of 10 to a team.

Ice Mario will do its best to make sure the points add up correctly. If the total is between 400 and 500, it will make sure the points add to 468, and if the total is between 900 and 1050, it will make sure the points add to 984. Outside this range, Ice Mario will check to see if the total is divisible by either 39 or 82. If everything is correct, the table message will be green. Otherwise, there will be one or more warning messages.

4. Logos

The logos manager.

!getlogo <teamName>
Send back the team logo sepcified by teamName if available.

!listlogos
Get a list of all the logos available.

!addlogo <teamName> [optional image link]
Adds a new logo. The clan tag should be one word without any spaces. Optionally, you can provide the logo link in optional image link if you have the url available. Otherwise, send the image when using the command. When logos are added, they will be immediately available, and they will eventually become a "built-in" logo. This just means they will be cropped, the background will be removed, and a team color will be assigned to it. Built-in logos should not be overwritten, but if needed, you can use !addlogoforce to add one.

!removelogo <teamName>
Removes a logo. Built-in logos shouldn't be removed normally. If a logo changes, use !addlogo to update it. If you really want to remove a logo, then you can use !removelogoforce.

!logo <team1> <team2>
Note: this command is deprecated, and !tablel should be used instead. This adds team logos to an already made table. Use a dash (-) to omit the logo for a team.

5. MKTC Lounge

Commands to get info from the MKTC Lounge spreadsheets. Note: the following commands will only work if enable-lounge-stats is set to true (default is true).

!mmr [optional names list]
Get the mmr of the command user or a list of names. An average mmr will also be calculated if there are two or more names. Here are some examples:
!mmr: Get your mmr. Make sure your nickname can be found in the spreadsheet.
!mmr player1: Get the mmr of player1.
!mmr player1, player2, player3, player4: Get the mmr values of all four players.

!stats [optional name]
Get the stats of the 10 most recent events of the command user. Specify optional name to find the name of someone else. This can either be their name or a member ping (@someone).

!stats# [optional name]
Get the stats of the command user for the lounge season specified using #. If optional name is provided, this will get the stats for them instead of the command user. For example, use !stats6 to get your season 6 stats.

!statsall [optional name]
Get all the stats of the command user of all seasons starting from season 1. If optional name is provided, this will get the stats for them instead of the command user. Note that this command may not find all the seasons since a name change between seasons can cause some to be missed.

!top [optional minRank] [optional maxRank]
Get the players in the specified rank range. If both options are omitted, the top 10 players will be shown. If one option is provided, then the 10 highest ranks from it are shown. The maximum rank range is 10.

!last# [optional name]
Get the stats of the command user, but only take into account the last n events specified in #. If optional name is provided, this will get the stats for them instead of the command user. For example, use !last15 to get your stats for the 15 most recent events.

!scoreboard or
!scoreboard # player1, player2, ..., player8
Generate the scoreboard for a mogi. Note: This command is only available in the MKTC lounge server and in servers where enable-mogi-gatherer is set to true.

!mmrlu
Get the mmr of the current mogi lu. This command is only available in the MKTC Lounge server.

6. Multiplayer Cups

!cups <time>
Get the cups and tracks that are live at a certain time. Example use cases:
!cups now - This will use the current time in UTC
!cups 8pm edt
!cups 7:00 am pdt
!cups 10/29 22:00 CEST

!stickycups
Initialize a sticky cups message. This will allow Ice Mario to update the image of the current cups once a minute. Ideally, this command should be used in a read-only Discord channel for members to look at without chatting. There can only be one stickycups message per server. To move an existing stickycups message to a new channel, use !stickycups in the new channel. To stop, simply delete the message. Note: you will need MANAGE_MESSAGES permission to use this command.

7. Ranked Scores

Get an estimate of the possible score on a track given your loadout.

im!score
Get an estimate of the score possible for your loadout. You should type im!score, and Ice Mario will then send a form for you to fill out. Copy the whole message, fill it out, then send it back. For kart rarity, n=normal, s=super, and h=high end.

im!ac
Get an estimate of the action count required to achieve a desired score based on your loadout. This is almost exactly like im!score, but you specify the points instead of Action Count.

8. Custom Commands

Custom commands manager for a server

!cca <command>, <value> or
!customcommandadd <command>, <value>
Add or update a custom command. There should be a comma separating command and value, and both of them can be more than one word. If you would like Ice Mario to send back an image instead of text, provide the image link to value. All custom commands will use "!" as the prefix. Here is an example of adding the command !invite:

!ccr <command> or
!customcommandremove <command>
Remove a custom command. For example, to remove the command !invite from above, use !ccr invite

!ccl or
!customcommandlist
List all the custom commands set in a server.

9. Mogi Gatherer

These commands are intended to behave the same as Mogibot, but geared specifically for MKT. Originally, these were created as a backup in case Mogibot ever went offline. Note: the following commands will only work if enable-mogi-gatherer is set to true (default is false). There are a few config options available that allows for greater flexability. Overall, Ice Mario will ping @here +4 when a mogi is half full, and it will ping @here +1 if there are 7 players. When there are 8 players, voting will start, and players will not be able to drop from the mogi. If a player joined 2 or more mogis and one fills, they will automatically be removed from the other mogis.

!start
Start a mogi.

!end
End a mogi.

!esn
End the current mogi, if it exists, and start a new mogi.

!c / !can / :pepec:
Join a mogi. After 20 minutes of inactivity, Ice Mario will ping the player to type something in the chat, and they will be removed after 25 minutes of inactivity. These can be adjusted in the server config settings.

!d / !drop / :peped:
Drop from the current mogi. You cannot drop once a mogi has gathered 8 players.

!da / !dropall
Drop from all mogi. You cannot drop once a mogi has gathered 8 players.

!l / !list
List all the players in the current mogi.

!r # / !remove #
Remove a player from the current mogi, where # is the player number shown after using !l. Only moderators can use this command, and they can use this command to remove someone even after a mogi has started.

!mllu / !mogilistlineup
List all the players in each mogi in the server.

!p / !#
Ping @here plus the number of players needed to start the mogi. # can be between 1 and 6.

!tag [optional text]
Tag the first 8 players in the mogi list. If optional text is provided, it will be added to the end of the member pings.

!votes
See what the vote tally is. This can only be used once voting has started.

^teams
Show the generated teams once voting has ended.

!mmrlu
Get the mmr of the current mogi lu. This command is only available in the MKTC Lounge server.

!teams
Randomize teams. More info can be found in the miscellaneous commands section here.

!scoreboard or
!scoreboard # player1, player2, ..., player8
Generate the scoreboard for a mogi. Note: This command is only available in the MKTC lounge server and in servers where enable-mogi-gatherer is set to true.

10. Miscellaneous commands

These are commands that can be individually enabled or disabled. Also, these commands are disabled by default, but they can be enabled by changing the server config settings.

!choose [optional number] item1, item2, ..., itemN
Randomly choose one or more items in the list of items provided. If optional number is omitted, Ice Mario will choose 1 item out of the list. Otherwise, Ice Mario wil choose optional number number of items. The items list can be separated using either a comma or a space. enable-choose must be set to true from the config settings for this to work. Here are some example use cases:

!choose banana, orange, mango, strawberry, pineapple: 1 item will be randomly chosen.
!choose 1 banana, orange, mango, strawberry, pineapple: 1 item will be randomly chosen.
!choose 2 banana orange mango strawberry pineapple: 2 items will be randomly chosen.

!teams # player1, player2, ..., playerN
Randomize n players into the number of teams specified in #. enable-randomize-teams must be set to true from the config settings for this to work. Here are some example use cases:

!teams 2 player1, player2, ..., player8: Randomly separate 8 players into two teams for a 4v4.
!teams 4 player1, player2, ..., player8: Randomly separate 8 players into four teams for a 2v2.

11. Server Configuration

These are all the server config options currently available, as well as commands to set them. Note: the following commands will only work if you have MANAGE_MESSAGES permission.

im!config
List all the current config options for the server.

im!config <key> <value>
Set the config option key with the value value. key is one of the options listed below, and they all use a dash (-) instead of a space. For example, to enable the !choose command, use im!config enable-choose true

im!resetconfig
Reset all the server config options to the default values. The default values are shown below.

The following is a list of all the config options (in blue) available as well as their default values (in orange):

Clan War Scheduler
  • enable-schedule (Default: true): Enable or disable all scheduler commands and reactions.
  • ping-one-hour-before (Default: true): If set to true, members that have reacted to the schedule will be pinged 1 hour before the scheduled event. If set to false, there will not be such a reminder.
  • ping-when-starting (Default: true): If set to true, members that have reacted to the schedule will be pinged when it is time for the scheduled event. If set to false, there will not be such a reminder.
  • delete-schedule-after-one-hour (Default: false): If set to true, schedules will automatically be deleted 1 hour after the scheduled event. If set to false, the schedule will not be deleted.
  • moderators-only-for-scheduler (Default: false): If set to true, only members with MANAGE_MESSAGES permission can create and edit schedule. If set to false, anyone in the server can do those things.
Clan War Score Tracker
  • enable-score-tracker (Default: true): Enable or disable all score tracker commands.
  • default-team-name (Default: Team1): Sets the default team name for your server. You can use this to set your team tag so that you don't have to set it everytime you use !startwar or !setteams.
Table Maker
  • enable-table-maker (Default: true): Enable or disable all table maker commands. Note that this does not have any effect for logo commands such as !addlogo or !getlogo.
Lounge
  • enable-lounge-stats (Default: true): Enable or disable all MKTC lounge commands such as !mmr and !stats.
Mogi
  • enable-mogi-gatherer (Default: false): Enable or disable all mogi commands.
  • inactive-minutes-until-drop (Default: 25): The number of inactive minutes before someone is dropped from the mogi.
  • minutes-to-ping-before-drop (Default: 5): The number of minutes before inactive-minutes-until-drop to ping someone to have them type something in the chat.
  • max-voting-minutes (Default: 2): The number of minutes to wait for people to vote on the mogi format after it has started. If the votes did not get 4 votes, then this will close the mogi voting once time is up.
  • should-ping-here (Default: true): Whether or not to ping @here.
Custom Commands
  • enable-custom-commands (Default: true): Enable or disable custom commands in your server.
  • allow-anyone-to-update-cc (Default: true): If set to true, anyone in the server can add, update, or delete custom commands. If set to false, only people with MANAGE_MESSAGES permission can do so.
Miscellaneous
  • enable-choose (Default: false): Enable or disable the !choose command.
  • enable-randomize-teams (Default: false): Enable or disable the !teams command to randomize teams for a mogi.
